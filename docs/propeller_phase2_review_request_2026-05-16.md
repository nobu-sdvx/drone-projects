## 4. ★レビュー要点(2 件)

### 4.1 【要点①】ブレード断面翼型 — NeuralFoil の最良は「薄キャンバ平板」

設計書 §4.4 は候補を S1223(1番手)/ E205 / 薄キャンバ平板 / AG12 としていた。
動作 Re 域(代表 Re=14,000)で NeuralFoil(xlarge)解析した L/D 最大値:

| 翼型 | L/D_max | α@LDmax |
|---|---|---|
| S1223 | 5.4 | 6.5° |
| E205 | 5.3 | 7.0° |
| AG12 | 11.3 | 3.5° |
| **薄キャンバ平板(NACA4402 近似)** | **17.4** | **5.5°** |

**実装側の方針(レビュー求む)**:
- **薄キャンバ平板系を採用** し、改善 5 項目①「翼型最適化」の具体的中身とする。
- 根拠:極低 Re(8–20k)では翼型は薄く低キャンバなほど剥離しにくく有利という教科書的
  事実。NeuralFoil の順位もこれと整合。3D プリントも容易。
- **ただし** Re<20k は NeuralFoil/XFoil の信頼度低下域(設計書 R3)。計算書では
  「相対比較で薄翼有利」を主たる根拠とし、L/D の絶対値には ±不確実性を明記する。

**レビュー質問**:
1. 「極低 Re では薄キャンバ平板が高キャンバ翼に勝る」という根拠で翼型を確定してよいか。
   審査会で工学的に説明できるか。
2. NeuralFoil の Re=14,000 での絶対 L/D(S1223=5.4 等)は明らかに低めだが、相対ランキング
   を信頼してよいか。XFLR5 等での追検証は Phase 3 送りで妥当か。

### 4.2 【要点②】設計書 §4.7 の推力計算に約 5.5 倍の算術誤差

設計書 §4.7 / §3.4 は「Crazyflie 45mm @19,860rpm の実測推力 = 0.016 N(1.65g重)」と
している。しかし UIUC 実測 CT から推力を計算し直すと:

```
T = CT × ρ × n² × D⁴
  = 0.1645 × 1.225 × (19860/60)² × 0.045⁴
  = 0.0906 N  ( = 90.6 mN = 9.2 g重 )
```

設計書 §4.7 の手計算「0.165×1.225×331²×0.045⁴ = 0.0164 N」は算術ミスで、正しくは
**0.0906 N**(手計算・QPROP・スクリプトで三重に確認)。
Crazyflie(総重量 ≈27g、4 発)が 1 発 9g 級の推力を出すのは物理的にも妥当。

**この訂正の影響(重要)**:

| | 設計書 §4.7(誤) | 訂正後 |
|---|---|---|
| Crazyflie 実測静推力 | 1.65 g重 | 9.2 g重 |
| 自作 36mm 単純スケーリング推力 | 0.83 g重 | **4.68 g重(45.9 mN)** |
| 必要推力 2.7g重(26.5mN)に対し | 30%(マージン薄い) | **174%(余裕あり)** |

→ 設計書 §4.7 の結論「推力マージンは薄い/必要推力に届かない懸念」は、この算術誤差に
基づくものだった。**訂正後は推力に十分な余裕がある。**

**交差検証(訂正後の数値の裏付け)**:
- QPROP 自作 36mm 静推力 @20,000rpm = 45.7 mN
- Crazyflie 実測 CT を D=36mm に換算 @20,000rpm = 37.6 mN
- → 同一 RPM で 21.5% 一致(独立手段で整合、低 Re ±20% 想定の範囲内)

**実装側の方針(レビュー求む)**:
1. §4.7 を訂正し、計算書の推力論証を 3 段で組み直す:
   - (a) 最大能力:40Krpm で静推力 184.7 mN ≈ POWERUP 公式 186mN
   - (b) 巡航必要推力 26.5 mN は V=7.5 で約 25,500 rpm にて到達(モーター上限 40K 内)
   - (c) UIUC 実測との交差検証で予測の信頼性を担保
2. **巡航動作 RPM の想定値を見直す**。設計書は 22,100rpm(ピッチ幾何 n=V/(P(1−slip))、
   slip=0.25 から逆算)としていたが、QPROP の推力釣り合いでは V=7.5 で必要推力 26.5mN
   到達は **約 25,500rpm**。slip=0.25 の仮定が小さすぎた。「巡航必要推力到達 RPM ≈ 25,500」
   へ修正、または併記する。
3. `requirements.md` / 設計書 v0.3 §4.7・§3.5 の数値修正(Nobu 承認後に実施予定)。

**レビュー質問**:
1. §4.7 の訂正(0.016N → 0.091N)と、それに伴う「マージン薄い→余裕あり」への結論変更は
   妥当か。見落としは無いか。
2. 巡航動作 RPM を 22,100 → 約 25,500 に見直す扱いは適切か。計算書ではどう書くべきか
   (「ピッチ幾何 RPM」と「推力釣り合い RPM」を区別して併記する案を検討中)。
3. 巡航点(22,100rpm・V=7.5)で QPROP 推力が 13mN(必要の 49%)なのは、想定巡航 RPM が
   低すぎたためと解釈してよいか。`thrust_vs_rpm_curves.png` のとおり 25,500rpm で必要推力に
   到達し、40K 上限に十分収まる。

## 5. レビュー観点チェックリスト

### 5.1 解析手法

1. QPROP(BEM ソルバ)+ 自作 BEM + UIUC 実測の 3 系統交差検証という構成は妥当か。
2. 自作 BEM と QPROP が設計点で 5.7% 一致(13.0 vs 13.8 mN)。これを「自作 BEM の妥当性
   確認」として計算書に書いてよいか。
3. 動作 Re 域はスクリプトで Re ≈ 2,400(根本)〜15,500(翼端付近)。設計書 §4.4 の
   「8k–20k」は中〜外翼の値で、根本はさらに低い。この扱いでよいか。

### 5.2 強度設計

4. SF(40K 最悪)= 3.9 は目標 4 をわずかに下回るが、実動作 27K/22K では 8.0/10.9。
   設計書 §5.4 と同じ「40K は最悪ケース、実動作で余裕」という扱いで妥当か。
5. 感度解析:m_blade=0.4g(最悪想定)でも 27K 動作で SF=6.3 を維持。天秤実測待ちの
   暫定値 0.3g の扱いとして十分か。

### 5.3 ①②の工学的妥当性(本書 §4)

6. §4.1【要点①】翼型を薄キャンバ平板系に確定する根拠は審査会で説明可能か。
7. §4.2【要点②】§4.7 算術誤差の訂正と推力論証の組み直しは妥当か。

### 5.4 計算書 §5 への展開

8. 図表 11 枚(指示書 §5.2)で「設計者として保証する」を示すのに過不足はないか。
9. 訂正後の推力余裕(必要の 174%)を踏まえ、計算書 §5 の結論はどうまとめるべきか。
