## 1. 前提と実施作業

### 1.1 機体構成(★ v2 追記:以降の「÷2」「1 モーター」の前提)

本機は推進モジュール **POWERUP 4.0 を 1 ユニット搭載**し、これは **2 基のマイクロモーターを
持つ双発構成**である(設計書 v0.3 §3.2)。操舵は左右モーターの差動推力で行う。
したがって:

- 自作プロペラは **左右同仕様 ×2 枚**(各モーターに 1 枚)。
- 「1 モーターあたり必要推力 = 全機抗力 ÷ 2」の「÷2」はこの双発構成に由来する。
- 強度・空力解析はすべて **モーター 1 基 + プロペラ 1 枚**を対象とする。

### 1.2 実施作業(Phase 2、2026-05-16)

1. **QPROP 1.22 を Windows に移植**(handoff §3〜§4)。動作確認テスト 4 本が macOS 結果と一致。
2. **Python スクリプト 3 本 + 共有設定モジュールを実装**(指示書 §3)。
3. **ブレード断面翼型を NeuralFoil で 4 候補スクリーニング**(指示書 優先1)。
4. **計算書 §5 用の図表 11 枚を生成**(指示書 §5.2)。

### 1.3 解析手法 — 検証の独立性(★ v2 修正)

3 つの数値手段を用いるが、**独立性の度合いが異なる**ことを明記する:

| 手段 | 位置づけ | 独立性 |
|---|---|---|
| QPROP 1.22(MIT Drela、BEM ソルバ) | 主たる数値 | — |
| 自作 BEM(ブレード要素運動量理論) | ソルバ実装の相互検証 | QPROP と **同じ NeuralFoil ポーラを入力**に使うため、翼型データ自体は検証していない |
| UIUC Crazyflie 実測データ | **唯一の独立な実測基準** | 翼型・解析モデルから完全に独立 |

→ QPROP↔自作 BEM の一致は「2 つの BEM 系ソルバ実装が同じ空力入力で整合する」ことを示す
(=ソルバ相互検証)。**翼型ポーラの妥当性を担保するのは UIUC 実測との照合のみ。**

## 3. 構造解析の数値整合(★ v2:注A 差し替え)

`propeller_structural_analysis.py` の出力は本書 §2 の構造数値とビット一致する
(SF 3.9/8.0/10.9、F_cf 71.1 N、σ 10.27 MPa、M_tip 0.220、ハブ SF 135)。

### 3.1 設計書 §5.5 との SF 差の真因(v1 注A の撤回と差し替え)

v1 では 27K/22K の SF が設計書 §5.5(8.9/12.9)とずれる理由を「RPM 丸め」と書いたが
**これは誤り**(応力 ∝ rpm² なので RPM を上げれば SF は下がり、向きが逆)。真因は以下:

**設計書 §5.5 は SF を 40K の値から rpm² で単純スケーリングしている。**

```
設計書値:  SF(22K) = 3.9 × (40000/22000)² = 12.9   ← 設計書 12.9 と完全一致
           SF(27K) = 3.9 × (40000/26600)² ≈ 8.8    ← 設計書 8.9 とほぼ一致
```

この単純スケーリングは「応力 σ 全体が rpm² に比例する」前提だが、実際は

```
σ_total = σ_T(遠心力、∝ rpm²) + σ_B(空気力曲げ、rpm に依存しない一定項)
```

設計書のスケーリングは一定項 σ_B を無視するため、**低 rpm で SF を過大評価**する。
σ_B/σ_total の比は 40K で 7.7%、22K で 21% と低 rpm ほど大きく、これが差を生む。
本スクリプトは σ_B を含めて各 rpm で一貫計算しており、27K/22K で設計書より低い
(=正しい)SF を与える。40K で両者一致(3.9)するのは σ_B 寄与が小さいため。

→ **本スクリプトの SF=3.9/8.0/10.9 を正とする。** 設計書 §5.5 の手計算 8.9/12.9 は
スケーリング近似の誤差であり、計算書では採用しない(設計書 §5.5 の数値修正を要する)。

### 3.2 σ_B の保守仮定(軽微指摘の反映)

曲げ応力 σ_B はモーター 1 基の **全推力 186mN を 1 枚のブレード**に与えて計算している
(設計書 §5.3 と統一)。実際は 2 枚で分担するため 1 枚あたり 93mN で、本評価は約 2 倍
保守的(安全側)。σ_B は σ_T の約 1/13 で支配項ではないため、この保守化が SF を
損なうことはない。計算書には「σ_B は 1 枚=全推力の保守仮定」と明記する。

## 4. ★レビュー要点(2 件)

### 4.1 【要点①】ブレード断面翼型 — NeuralFoil の最良は「薄キャンバ平板」

設計書 §4.4 は候補を S1223(1番手)/ E205 / 薄キャンバ平板 / AG12 としていた。
動作 Re 域(代表 Re=14,000)で NeuralFoil(xlarge)解析した L/D 最大値:

| 翼型 | L/D_max | α@LDmax |
|---|---|---|
| S1223 | 5.4 | 6.5° |
| E205 | 5.3 | 7.0° |
| AG12 | 11.3 | 3.5° |
| **薄キャンバ平板(NACA4402 近似)** | **17.4** | **5.5°** |

**実装側の方針(レビューで確認済み・確定)**:
- **薄キャンバ平板系を採用**し、改善 5 項目①「翼型最適化」の具体的中身とする。
- 根拠:極低 Re(8–20k)では翼型は薄く低キャンバなほど剥離しにくく有利という
  教科書的事実。NeuralFoil の順位もこれと整合。3D プリントも容易。
- **計算書での書き方**:絶対 L/D 値(flat plate 17.4 等)は楽観的で信頼度が低い
  (Re<20k は NeuralFoil/XFoil の信頼度低下域、設計書 R3)。よって計算書では
  「**相対順位 + 剥離の物理**」で論じ、L/D 絶対値には ±不確実性を明記する。
- **不確実性の波及**(軽微指摘の反映):選定翼型のポーラは QPROP/BEM の推力数値
  すべての空力入力である。翼型ポーラの不確実性は推力予測全体に波及するため、
  UIUC 実測との照合(§4.2)が信頼性担保の要となる。

第1回レビュー回答:物理的根拠は妥当、審査会で説明可。XFLR5 追検証は Phase 3 送りで可。

### 4.2 【要点②】設計書 §4.7 の推力計算に約 5.5 倍の算術誤差(★ v2:論証を修正)

設計書 §4.7 / §3.4 は「Crazyflie 45mm @19,860rpm の実測推力 = 0.016 N(1.65g重)」と
している。しかし UIUC 実測 CT から推力を計算し直すと:

```
T = CT × ρ × n² × D⁴
  = 0.1645 × 1.225 × (19860/60)² × 0.045⁴
  = 0.0906 N  ( = 90.6 mN = 9.2 g重 )
```

設計書 §4.7 の手計算「= 0.0164 N」は算術ミスで、正しくは **0.0906 N**(手計算・QPROP・
スクリプトで三重確認。第1回レビューでも独立計算で確認済み)。Crazyflie(総重量 ≈27g、
4 発)が 1 発 9g 級の推力を出すのは物理的にも妥当。

#### 4.2.1 算術訂正の事実

| 量 | 設計書 §4.7(誤) | 訂正後(正) |
|---|---|---|
| Crazyflie 45mm @19,860rpm の静推力 | 0.016 N(1.65 g重) | **0.0906 N(9.2 g重)** |

なお設計書 §4.7 はもう 1 点、「前進飛行で静推力の 2〜3 倍が出る」とするが、これも
**固定ピッチ・固定 RPM のプロペラでは誤り**(前進比 J が増えると推力は減少する)。
ただし算術訂正後は静推力ベースで十分な推力が示せるため、この誤った前進飛行論証は
不要であり、計算書では使わない。

#### 4.2.2 推力充足性の論証(★ v2:「174%」を廃し 3 段論証に)

v1 は「単純スケーリング推力 45.9mN ÷ 必要 26.5mN = 174% 余裕」と書いたが、これは
**静推力(J=0)を前進飛行(V=7.5)の必要推力で割った林檎と蜜柑の比**であり不適切
(第1回レビュー指摘3)。45.9mN は静推力換算値、26.5mN は巡航必要推力で、次元が違う。

計算書では以下の **3 段**で推力充足性を示す(単一比率は使わない):

1. **最大能力**:40,000rpm での静推力は **184.7 mN ≈ POWERUP 公式 186mN**。
   モーターの推力能力に問題はない。
2. **巡航動作点**:巡航必要推力 26.5 mN(V=7.5)は QPROP 推力曲線上で
   **約 25,500 rpm** にて到達する(§3.1 図 `thrust_vs_rpm_curves.png`)。
   これはモーター上限 40,000rpm に十分収まる。
3. **交差検証**:自作 36mm の QPROP 静推力と Crazyflie 実測 CT を D=36mm に換算した
   値は、同一 20,000rpm で 45.7 vs 37.6 mN(乖離 21.5%)と整合。低 Re ±20% 想定の
   範囲内で、推力予測が実測トレンドと矛盾しないことを示す。

#### 4.2.3 巡航動作 RPM の定義統一(★ v2:レビュー指摘5)

文書内で 2 つの RPM が混在していたため、定義を以下に統一する:

| 呼称 | 値 | 定義 | 用途 |
|---|---|---|---|
| ピッチ幾何 RPM | 22,100 | n = V/(P·(1−slip))、slip=0.25 | 設計書 §3.5 の凍結値 |
| 推力釣り合い巡航 RPM | 約 25,500 | QPROP で必要推力 26.5mN に到達する rpm @V=7.5 | **物理的な巡航動作点** |

slip=0.25 の仮定が小さすぎ、ピッチ幾何 RPM は実動作点より低い。**計算書では
「推力釣り合い巡航 RPM ≈ 25,500」を巡航動作点として採用**し、ピッチ幾何 RPM 22,100 は
参考(下限推定)として併記する。

構造三点評価(40K/27K/22K)は設計書の凍結 rpm で実施しているが、巡航点を 25,500rpm
とした場合の SF は **約 8.6**(22,100rpm の SF=10.9 と 26,600rpm の SF=8.0 の中間)。
いずれにせよ巡航時 SF ≥ 8 を確保する。→ 設計書 §3.5 の巡航 RPM 22,100 は要見直し
(Nobu 承認後に `requirements.md`/設計書を修正)。

## 5. レビュー観点チェックリスト

### 5.1 解析手法

1. §1.3 の「QPROP↔BEM は 2 ソルバ相互検証、UIUC が唯一の独立実測基準」という
   位置づけは妥当か。計算書でもこの書き分けを徹底すべきか。
2. 自作 BEM と QPROP の設計点一致(13.03 vs 13.77 mN、乖離 5.7%)は、§1.3 を前提に
   「2 ソルバ実装の相互整合」として書く。これで過大主張は無いか。
3. 動作 Re 域はスクリプトで Re ≈ 2,400(根本)〜15,500(翼端付近)。設計書 §4.4 の
   「8k–20k」は中〜外翼の値。この扱いでよいか。

### 5.2 強度設計

4. SF(40K 最悪)= 3.9 は目標 4 をわずかに下回る。40K はモーター到達可能 rpm なので、
   計算書では「到達可能な最悪ケースで SF=3.9、実動作 25.5K で SF≈8.6」と
   **正直に併記**する方針。これで妥当か。
5. σ_B の保守仮定(1 枚=全推力 186mN、実際は分担で 93mN)を明記する扱い(§3.2)で
   十分か。
6. 感度解析:m_blade=0.4g(最悪想定)でも 27K 動作で SF=6.3 を維持。天秤実測待ちの
   暫定値 0.3g の扱いとして十分か。

7. §4.1【要点①】薄キャンバ平板系への確定を「相対順位 + 剥離の物理」で論じる方針は
   審査会で説明可能か。
8. §4.2【要点②】算術訂正・3 段論証・RPM 定義統一は妥当か。

### 5.4 計算書 §5 への展開
