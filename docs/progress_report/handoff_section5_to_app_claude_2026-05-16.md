## 2. アプリ版 Claude にやってほしいこと

### 2.1 §5 を全体計算書に組み込む

- §5 を §1〜§9 の通し概略計算書に統合する。
- 既存の他章ドラフト:`docs/calculation_doc_section2_mainwing.md`(主翼)、`docs/section3_tail_design.md`(尾翼)。章番号・見出し体裁・用語・単位表記を全章で揃える。

### 2.2 図の配置

- 図5-1〜5-9 を §5 本文の `**図5-X** …` の位置に貼る。
- 図中の文字が小さい場合はキャプションで補足する。

### 2.3 審査会プレゼン資料(スライド)

- §5 の要点をスライド数枚にまとめる:設計値 → 強度・推力の数値 → 要件適合表。
- 構成は「条件を満たすことの保証」を主役にする(選定理由は 1〜2 文で簡潔に)。

### 2.4 最終推敲・想定問答

- 審査会の聴衆(教員)向けに文章トーンを最終調整。
- **想定問答を準備する**。特に結論表で △ にした 2 件は質問が来る可能性が高い:
  1. 「40,000 rpm で SF=3.9 と目標 4 を下回るが大丈夫か」
     → 40,000 rpm は全開・静止の過渡的極限。実動作域(≤約25,100 rpm)では SF≥8.8。ブレード質量の天秤実測と印刷方向の指定で詰める、と説明。
  2. 「翼型厚さ 2% は 3D プリントできないのでは」
     → そのとおり。2% は空力上の理想値。実機は印刷可能厚(0.4〜0.8mm)に従う。本概略設計は『極低 Re では薄翼が有利』という相対的結論を根拠とし、印刷可能厚での再評価は詳細設計で行う、と説明。

## 3. アプリ版 Claude がやらないこと(Claude Code 側に残す作業)

- **解析スクリプトの再実行・数値の作り直し** → 数値変更が要るなら Claude Code に戻す。
- **`requirements.md` / 設計書 v0.3 の数値訂正**(巡航 RPM 等の確定値書き換え)→ リポジトリ操作。Nobu の承認を得たうえで Claude Code 側で実施。
- **ブレード質量の天秤実測**(Nobu の実作業)→ 実測後、感度解析を Claude Code で確定。
